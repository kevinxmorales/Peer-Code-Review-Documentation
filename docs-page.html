Introduction Last updated: 2019-06-01

The Peer Code Review Platform is a web application built to help SUNY Oswego students become better programmers. The platform consists of a React frontend, Node JS and Spring Boot web services, and Postgres SQL database.

Installation

This section will cover how to get started with development on the platform. We'll cover setting up local development environment, running the application locally, and deploying to production.

Prerequisites

The following must be installed to develop locally.

  • Java 15
  • Node
  • Docker
  • Git
  • Maven
  • Yarn

Tip

If you are developing on Mac, it is highly recommended to install everything with Homebrew.

Warning

Running three docker containers simultaneously is a computationally intense process. It is highly recommended that the development machine have at least 8GB of RAM, but 16GB would be much better.

It is recommended that a local directory is made where all repos are cloned into subdirectories. Below is an example of what the development set up could look like.

Get the source code

Source code for the platform is split into three git repositories. These repos are private and require access permissions to view or clone.

  • Frontend: https://github.com/OswegoCS/PCR-Frontend
  • Backend: https://github.com/OswegoCS/PCR-backend
  • Database: https://github.com/OswegoCS/PCR-database
  • Documentation: https://github.com/OswegoCS/PCR-documentation

It is recommended that a local directory is made where all repos are cloned into subdirectories. Below is an example of what the development set up could look like.

Run the Database

While in the database project, you should see two files. A SQL dump file and a Dockerfile. The dump file holds data that will create the schema of the database.

To build the Docker image of the database run docker build -t pgdb .

To run this image as a container run docker run -p 5432:5432 -t -d -e POSTGRES_PASSWORD=postgresql pgdb

This Postgres Docker container should be running on port 5432. Use a tool like PgAdmin or TablePlus to view the database tables.

Running the Spring Boot REST API

While in the backend project, there should be a Dockerfile in the top level of the directory.

Use Maven to compile the backend code. Run mvn clean install. This command will compile the code to an executable JAR file located in the target directory.

To build the Docker image of the backend run docker build -t pgdb .

To run this image as a container run docker run -p 5432:5432 -t -d -e POSTGRES_PASSWORD=postgresql pgdb

This Postgres Docker container should be running on port 5432. Use a tool like PgAdmin or TablePlus to view the database tables.

Launch the User Interface

Local development with Yarn

While in the frontend project, there should be a Dockerfile in the top level of the directory.

Running the React app in development mode will allow you to see error traces that would be unavailable in production. Run yarn install. This command will pull all the dependencies needed to run the project. Once that finishes, run yarn dev to run the react app. The UI will now be accessible on port 3000.

Local Development with Docker

Eventually, working changes will need to be tested in a Docker container

To build the Docker image of the frontend run docker build -t client .

To run this image as a container run docker run -p 5432:5432 -t -d -e client . This React Docker container should be running on port 80.

Deploying Changes to Production

Use these commands to build Docker images of the respective platform components

React: docker build -t kevinmorales/js --platform linux/amd64 .

Postgres: docker build -t kevinmorales/pgdb --platform linux/amd64 .

Java Spring Boot: docker build -t kevinmorales/spring --platform linux/amd64 .

Warning

When running these commands you must be in the same directory as the Dockerfile. The period at the end denotes that the Dockerfile is in your current working directory.

Danger

The --platform linux/amd64 part of the command is important. The moxie servers are running Ubuntu Linux on AMD 64 architecture. Since development may occur on Windows, Mac OS, or any number of Linux distributions, all of which may be running Intel, AMD, or M-series architecture, this flag ensures that the finished docker container is compatible with the moxie servers. For more information on multi-platform docker builds, read the Docker Documentation.
